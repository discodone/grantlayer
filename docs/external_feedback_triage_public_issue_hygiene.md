# GL-183 External Feedback Triage / Public Issue Hygiene

## Feedback Categories

Blocker, confusing-docs, broken-quickstart, missing-example, product-question,
security-concern, production-readiness-concern, feature-request, and
non-scope-later.

## Proposed Labels

feedback, docs, quickstart, security-review-needed, production-readiness, and
needs-triage are the minimum labels for first external feedback.

## Severity

Severity values are critical, high, medium, low, and info.

## Intake Workflow

Classify category, assign severity, check safety constraints, decide public
issue vs private security advisory, record source and date, and decide next
action.

## Security Handling

Security-sensitive reports go to GitHub Security Advisories. Public issues must
not include secrets, exploit details, customer data, or private institutional
records.

## Explicit Statements

No GitHub push was performed. No visibility change was performed. Internal repo was not pushed directly to GitHub. No GitHub API label changes were performed.

## Non-Goals

No production SaaS promise, no tenant/workspace isolation claim, no customer
onboarding, and no SLA commitment are made by this issue.
