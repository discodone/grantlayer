⚠ This section intentionally corrupts a grant to demonstrate signature verification. No real data is affected beyond this local demo database.

The action will use the tampered role so the policy engine finds the grant — then the signature check catches the manipulation.

Why this matters A signed grant cannot be silently modified. Every grant is signed with Ed25519 over 9 immutable fields (id, subjectId, role, action, resource, validFrom, validUntil, createdBy, reason).

When this demo changes role directly in the database without re-signing, the stored payloadHash no longer matches the current fields. The backend catches this before approving any action.

Expected result approved: true before tamper
approved: false after tamper
